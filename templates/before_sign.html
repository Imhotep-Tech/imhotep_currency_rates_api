1. Authentication & Base URL

All requests to the Imhotep Currency Rates API require your API key as a path parameter. There are no additional headers or authorization prefixes needed.

Base URL
https://imhotepexchangeratesapi.pythonanywhere.com
Security Reminder Do not embed API keys directly in client-side applications (like frontend web applications or apps) as they can be easily extracted. Instead, proxy currency requests through your own backend server.
GET

/latest_rates/{api_key}/{base_currency}

Returns the current exchange rates relative to the requested base_currency. Rate updates are fetched from third-party APIs daily.

Path Parameters

Parameter Type Required Description
api_key string Yes Your unique API key or guest credentials
base_currency string Yes The reference currency code (e.g. USD, EUR)

Code Samples

// Fetch latest exchange rates
const apiKey = 'YOUR_API_KEY';
const url = `https://imhotepexchangeratesapi.pythonanywhere.com/latest_rates/${apiKey}/USD`;

fetch(url)
  .then(res => res.json())
  .then(data => {
    console.log('Base currency:', data.meta.base_currency);
    console.log('Rates:', data.data);
  })
  .catch(err => console.error(err));

Response Example (JSON)

{
  "meta": {
    "base_currency": "USD",
    "last_updated_at": "2026-05-30T18:00:00"
  },
  "data": {
    "EUR": 0.92,
    "GBP": 0.79,
    "JPY": 156.45,
    "CAD": 1.37,
    "AUD": 1.51,
    "EGP": 47.30
  }
}
GET

/convert/latest_rates/{api_key}/{base_currency}/{target_currency}/{amount}

Converts an exact amount of money from base_currency into target_currency using daily updated exchange rates.

Path Parameters

Parameter Type Required Description
api_key string Yes Your unique API key
base_currency string Yes The currency code to convert from (e.g. USD)
target_currency string Yes The currency code to convert to (e.g. EUR)
amount integer Yes An integer value to convert

Code Samples

// Convert currency amount
const apiKey = 'YOUR_API_KEY';
const url = `https://imhotepexchangeratesapi.pythonanywhere.com/convert/latest_rates/${apiKey}/USD/EUR/100`;

fetch(url)
  .then(res => res.json())
  .then(data => {
    console.log(`Converted: ${data.meta.amount} ${data.meta.base_currency} = ${data.data.converted_amount} ${data.meta.target_currency}`);
  })
  .catch(err => console.error(err));

Response Example (JSON)

{
  "meta": {
    "base_currency": "USD",
    "target_currency": "EUR",
    "amount": 100,
    "last_updated_at": "2026-05-30T18:00:00"
  },
  "data": {
    "conversion_rate": 0.924,
    "converted_amount": 92.40
  }
}

4. Standard Error Schema

All API failures return a unified JSON payload representing the error structure alongside standard HTTP status codes. 

{
  "error": {
    "code": 401,
    "message": "Invalid API key"
  }
}

HTTP Status Code Reference

Status Code Error Type Description
400 Bad Request Malformed parameters or currency codes that are not supported.
401 Unauthorized Missing or invalid API key credentials.
429 Too Many Requests Limit exceeded. Currently, the rate limit is set at 30 requests per minute.
503 Service Unavailable External rates feed failed and there was no stale cached data available.

5. Graceful Cache Fallback

In case the external API feed is down, our API will continue serving historical currency records stored locally, outputting a warning object showing data age. 

{
  "meta": {
    "base_currency": "USD",
    "last_updated_at": "2026-05-28T18:00:00"
  },
  "warning": {
    "message": "Using stale data as fallback. External API could not be reached.",
    "data_age": "2 days old"
  },
  "data": {
    "EUR": 0.92,
    "GBP": 0.79
  }
}
